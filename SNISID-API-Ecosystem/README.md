# SNISID National API Ecosystem & Interoperability

## Repository Structure
- `API-Gateway/`: National Gateway configurations (Kong/APISIX).
- `Service-Mesh/`: Internal service communication and security (Istio).
- `Registry/`: National API Catalog and metadata.
- `Interoperability/`: Standards and protocols (REST, gRPC, JSON).
- `Security/`: Zero Trust framework, OIDC, and mTLS.
- `Connectors/`: Inter-agency integration points (ONI, DGI, etc.).
- `Events/`: Event-driven architecture (Kafka) and event schemas.
- `Developer-Portal/`: Documentation and SDK management.
- `Governance/`: API lifecycle and approval policies.
- `Observability/`: Monitoring, Logging, and Tracing (LGTM stack).
- `Resilience/`: Offline and error-handling models.
- `KPIs/`: Performance measurement metrics.
- `Testing/`: Automated validation suites.
- `Runbooks/`: Operational procedures for incident response.
