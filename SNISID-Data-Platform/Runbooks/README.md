# Runbooks Data SNISID

## Objectif
Industrialiser les opérations data afin qu'elles soient répétables, documentées et auditables.

## Runbooks inclus

| Runbook | Description |
|---|---|
| [Data corruption recovery](./Data_Corruption_Recovery.md) | Recovery corruption données |
| [Pipeline failure resolution](./Pipeline_Failure_Resolution.md) | Résolution pipeline en échec |
| [Data breach containment](./Data_Breach_Containment.md) | Containment fuite de données |

## Règles générales

- Ouvrir incident avec sévérité.
- Assigner Incident Commander.
- Préserver preuves et logs.
- Communiquer selon matrice officielle.
- Documenter timeline, impacts, actions.
- Réaliser post-mortem sans blâme.
- Mettre à jour contrôles préventifs.
