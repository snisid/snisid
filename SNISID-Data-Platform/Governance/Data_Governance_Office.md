# Data Governance Office SNISID

## Mission
Gouverner les données nationales en garantissant ownership, classification, qualité, conformité, accès, rétention et auditabilité.

## Structure

| Rôle | Responsabilités |
|---|---|
| Chief Data Officer national | Stratégie, arbitrage, politique nationale |
| Data Owners | Responsabilité officielle des datasets |
| Data Stewards | Qualité, définitions, classification, résolution conflits |
| Data Custodians | Opérations techniques, stockage, pipelines |
| Security Officer | Protection, accès, chiffrement, incidents |
| Privacy/Legal Officer | Consentement, finalité, conformité, legal hold |
| Audit Officer | Contrôles, preuves, rapports |

## Responsabilités clés

| Fonction | Description | Obligation |
|---|---|---|
| Data ownership | Responsabilité officielle | Aucun dataset sans owner |
| Classification | Sensibilité et criticité | Classification à la création |
| Retention | Conservation | Politique associée obligatoire |
| Access governance | Permissions | RBAC/ABAC + justification |
| Compliance | Réglementation | Revue périodique |
| Data quality | Exactitude et cohérence | Seuils automatisés |
| Metadata governance | Catalogue | Aucune donnée orpheline |

## Comité de gouvernance

- Réunion mensuelle pour données standard.
- Réunion hebdomadaire pour domaines critiques.
- Procédure d'urgence pour incident data/security.
- Droit de blocage sur publication Gold non conforme.

## Workflow d'approbation d'un dataset

1. Demande de création dataset.
2. Désignation owner/steward.
3. Classification et sensibilité.
4. Définition contrat de données.
5. Règles qualité et rétention.
6. Politique d'accès.
7. Enregistrement metadata/lineage.
8. Validation Security/Legal si sensible.
9. Publication contrôlée.
