# National Resilience Testing Program

## 1. Objectif
Vérifier concrètement la résilience nationale SNISID par des tests récurrents, mesurés et auditables.

## 2. Tests
| Test | Description | Fréquence |
|---|---|---:|
| DR failover drills | bascule vers secondary/DR | mensuel partiel, semestriel complet |
| Full recovery exercises | reconstruction complète critique | semestriel |
| Offline survival drills | opérations sans Internet national | trimestriel |
| Crisis command simulations | activation NRCC/inter-agences | trimestriel |

## 3. Calendrier annuel
| Trimestre | Exercices |
|---|---|
| T1 | tabletop crise nationale, restore B0, offline verification |
| T2 | failover partiel, ransomware drill, power autonomy |
| T3 | full recovery, satellite/radio, offline enrollment régional |
| T4 | exercice national multi-agences, audit KPI, mise à jour doctrine |

## 4. Mesures
Temps détection, activation NRCC, failover, RTO/RPO réel, succès restauration, disponibilité P0/P1, latence communications, écarts runbook.

## 5. Rapport post-exercice
```text
Nom/date/scénario/objectifs/participants
Chronologie
RTO/RPO mesurés
Réussites et écarts
Risques résiduels
Actions correctives, responsables, échéances
Validation finale
```

## 6. Cycle
Tester → Mesurer → Corriger → Automatiser → Re-tester → Certifier.
